# Set Tools — Backend (Phase 2)

Supabase backend that replaces localStorage with a real database, syncs state across devices (DP laptop ↔ kiosk tablet), and proxies the Anthropic API so the key never lives in the browser.

## Why Supabase?

- Free tier covers a small production easily (500 MB DB, 5 GB bandwidth/mo)
- Postgres + realtime subscriptions out of the box → kiosk auto-updates when DP uploads a call sheet
- Edge Functions run your API proxy without managing servers
- Auth: one production = one workspace, devices join with a short code

## One-time setup

1. Sign up at https://supabase.com (free)
2. Create a new project — pick a region close to you, save the DB password somewhere safe
3. Install CLI: `npm install -g supabase`
4. From the project root, link your local config to the cloud project:
   ```bash
   supabase login
   supabase link --project-ref <your-project-ref>
   ```
   (The project-ref is in your Supabase dashboard URL: `app.supabase.com/project/<ref>`)
5. Push the schema:
   ```bash
   supabase db push
   ```
6. Set the Anthropic API key as a secret (never ends up in code):
   ```bash
   supabase secrets set ANTHROPIC_API_KEY=sk-ant-...
   ```
7. Deploy the Edge Function:
   ```bash
   supabase functions deploy claude-proxy
   ```

## What lives where

```
backend/
├── README.md                              ← you are here
├── supabase/
│   ├── config.toml                        ← project config (generated by `supabase init`)
│   ├── migrations/
│   │   └── 0001_initial_schema.sql        ← all tables + RLS policies
│   └── functions/
│       └── claude-proxy/
│           └── index.ts                   ← Anthropic API proxy
```

## Frontend integration (sketched, do in Phase 3 or earlier)

Replace the direct `fetch('https://api.anthropic.com/...')` calls in your HTML files with a call to your Edge Function:

```js
// before:
fetch('https://api.anthropic.com/v1/messages', { headers: {'x-api-key': key, ...} })

// after:
fetch('https://<project-ref>.supabase.co/functions/v1/claude-proxy', {
  headers: { 'Authorization': 'Bearer ' + supabaseSession.access_token },
  body: JSON.stringify({ model, messages, system })
})
```

The Edge Function reads the key from the secret on the server side and forwards the request to Anthropic. Your browser never sees the key.
